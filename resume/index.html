项目经验

志愿者派遣系统

投入人数：100人左右

使用技术：Java + jsp + javaScript + Struts + Oracle

开发工具：Eclipse，Tomcat

时间：开发阶段2007.8 ~2008.11,维护阶段2008.11~2013.9

项目描述：

项目为一个Web系统，用于志愿者派遣流程的管理。系统最终发布到Unix系统上，主要用于日本政府的志愿者派遣工作。该系统针对志愿者本人，有如下功能：报名，培训，修改个人信息，各种结果查询，打印各种文档等。针对管理人员有如下功能：审批，安排志愿者，设置各种初始数据，修改志愿者信息，给志愿者发放补贴等功能。整个系统的模块涉及到志愿者从报名，派遣，回国，发放薪水等整个流程。

工作责任描述：

  1. 修改HTML网页到jsp网页，用struts标签或者自己开发的标签代替
  2. 在jsp网页，通过JavaScript调用Java程序。
  3. 使用Form+Action+Logic的架构来开发后台Java程序
  4. 通过validator来对画面的操作和输入数据进行相关检查和限制
  5. 配置一些Struts的配置文件，如：default_values.properties, log4j-config.xml
  6. 根据基本设计来编写详细设计
  7. 根据详细设计设计测试用例并实施测试用例
  8. 维护后期，掌握整个系统的业务流程。维护各种文档，为客户准备修改的代码。
  9. 通过Ant脚本编译程序并打包，用Tomcat发布程序
  10. Review其他人开发的代码
  11. 后期维护阶段负责管理3人维护团队的管理工作，负责与客户沟通，负责给团队成员分配工作和检查进度
